# Processed Data Artifacts

## Overview
This folder contains processed data generated during the data preprocessing and sequence engineering stages.

## Files

### Small Files (Safe to Commit to Git)
- **`class_weights.json`** - Class weights for handling imbalanced data (computed from training set only)
- **`feature_names.txt`** - List of engineered features used in the model
- **`sequence_scaler.pkl`** - StandardScaler fitted on training sequences for feature normalization

### Large Files (Generated Locally, NOT in Git)
These files are excluded from git to avoid repository bloat. They are **automatically generated** by running the notebooks:

- **`cleaned_data.parquet`** (261 MB)
  - Clean, preprocessed dataset with train/val/test splits
  - Generated by: `notebooks/data_preprocessing.ipynb`
  
- **`cleaned_data_with_identifiers.csv`** (1.3 GB)  
  - CSV version of cleaned data (same as parquet, for inspection)
  - Generated by: `notebooks/data_preprocessing.ipynb`

- **`sequences_normalized.npz`** 
  - LSTM-ready sequences (normalized, time-ordered)
  - Generated by: `notebooks/sequence_engineering_lstm_design.ipynb`

## How to Regenerate Files

### Step 1: Run Data Preprocessing
```bash
cd notebooks
jupyter notebook data_preprocessing.ipynb
```
This will generate:
- `cleaned_data.parquet`
- `cleaned_data_with_identifiers.csv`
- `class_weights.json`
- `feature_names.txt`
- `scaler.pkl`

### Step 2: Run Sequence Engineering
```bash
jupyter notebook sequence_engineering_lstm_design.ipynb
```
This will generate:
- `sequences_normalized.npz`
- `sequence_scaler.pkl`

## Important Notes

✅ **Do not** commit `.parquet` or `.csv` files to git  
✅ **Do commit** `*.json` and `.txt` files (small and necessary)  
✅ **Do commit** `.pkl` scaler files (needed for model inference)  
✅ **Never** manually edit processed data files - always regenerate via notebooks  
✅ **Always** run notebooks in order (preprocessing → sequence engineering → model training)

## File Sizes
- Total processed data (locally): ~1.6 GB
- Files tracked in git: < 1 KB
- This approach keeps the repository lean while preserving reproducibility

## For Team Members
1. Clone the repository
2. Install Python packages: `pip install -r requirements.txt`
3. Run `notebooks/data_preprocessing.ipynb` to generate processed data
4. Run `notebooks/sequence_engineering_lstm_design.ipynb` to generate sequences
5. You now have all data needed for model training!
